# Pulse Dovetail Clone: Theme Analysis & Visualization Guide

This document provides a deep dive into the **Theme** system—the core intelligence layer of the Pulse Dovetail Clone. It explains how raw data signals are transformed into synthesized insights and how those insights are presented to users.

---

## 1. What is a Theme?
In this system, a **Theme** is a synthesized insight that represents a cluster of similar feedback "signals" (Slack messages, CSV rows, or Markdown segments). 
- **Goal:** To reduce noise and provide a high-level understanding of what users are saying.
- **Hierarchy:** `Topic` (Category) > `Theme` (Insight) > `Signal` (Evidence).

---

## 2. The Analysis Pipeline

### A. Triggering Analysis
Analysis can be triggered in three ways:
1.  **Manual Refresh:** Users click "Analyze" in the dashboard.
2.  **Specific Selection:** Users select specific messages to "Analyze Selection."
3.  **Cron Job:** A scheduled process (`/api/channels/[id]/analyze`) runs periodically to process new incoming signals.

### B. AI Prompting Logic (`analyzeThemes`)
The system uses the `analyzeThemes` function in `src/lib/ai.ts`. The prompt instructs the LLM to:
- Group messages into **2-8 distinct themes**.
- Extract themes directly if they are explicitly mentioned.
- **Contextual Awareness:** The LLM receives `ai_context` (provided by the user in channel settings) to focus on specific business goals (e.g., "Look for bug reports and UI friction").
- **High-Density Handling:** If a single large document (like an interview transcript) is provided, the AI is instructed to extract more granular themes to cover the data density.

### C. Sentiment & Data Mapping
For every theme the AI extracts, it returns a list of `message_ids`. The system then:
1.  **Calculates Sentiment:** Sums the sentiment (Positive, Negative, Neutral) of all linked signals.
2.  **Maintains Continuity:** Checks if a theme with the same name already exists. If it does, it updates the summary and counts; if not, it creates a new record.
3.  **Links Evidence:** Populates the `data_point_themes` table to create a permanent link between the theme and its source signals.

## 3. Visualization: How Themes are Presented

### A. The Theme Card (`ThemeCard.tsx`)
The primary view for themes is a high-impact card designed for quick scanning:
- **Sentiment Spectrum:** A multi-colored progress bar at the bottom showing the ratio of green (positive), gray (neutral), and red (negative) signals.
- **Synthesis Summary:** A 3-line AI-generated summary of the theme's core insight.
- **Metadata Badges:**
    - **Signal Count:** Total number of messages supporting this theme.
    - **Manual vs. AI:** Indicates if the theme was generated by the system or manually created/edited by a user.
    - **Pinned:** Highlights high-priority themes.

### B. The Deep Dive (`ThemeDrawer.tsx`)
Clicking "Analyze Messages" opens a side drawer for research:
- **Volume Trend:** A `LineChart` (Recharts) visualizing the frequency of this theme over time using daily snapshots.
- **Trend Indicators:** A percentage change (e.g., "+24% vs last week") with color-coded arrows (Rising/Falling).
- **Evidence List:** A scrollable list of the actual raw signals (`MessageCard.tsx`) so researchers can verify the AI's conclusions.

---

## 4. Theme Management Features

### A. Merging Themes
Researchers can merge two themes together if the AI has created redundant clusters. This preserves the links to all underlying signals while consolidating the name and summary.

### B. Daily Snapshots
Every time analysis runs, the system records a snapshot in the `theme_snapshots` table:
- **Captured Data:** Date, Theme ID, Signal Count, and Sentiment Breakdown.
- **Purpose:** This enables the historical trend visualization and anomaly detection (detecting sudden spikes in specific feedback).

### C. Topic Categorization
Themes are automatically grouped into **Topics** (e.g., "Onboarding", "Billing", "Mobile App"). This allows users to filter their entire dashboard by broad categories while still seeing specific thematic insights.
